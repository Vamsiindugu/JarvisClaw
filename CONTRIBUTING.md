## Pull Request Process

1. **Fork** the repository
2. **Create a branch**: `git checkout -b feature/your-feature-name`
3. **Make changes** following our conventions
4. **Test** your changes
5. **Commit** with clear messages: `git commit -m "Add: your feature description"`
6. **Push**: `git push origin feature/your-feature-name`
7. **Open PR** with description of changes

## Conventions

### File Naming
- Use `UPPERCASE.md` for core files
- Use `lowercase.md` for templates
- Use `kebab-case.md` for auxiliary files

### Markdown Style
- Use `#` for title, `##` for sections, `###` for subsections
- Use tables for structured data
- Use code blocks with language hints

### Persona Files
When creating persona templates:
1. Include personality matrix
2. Include core directives
3. Include addressing protocol
4. Include communication style

## Code of Conduct

- Be respectful and inclusive
- Welcome newcomers
- Accept constructive criticism
- Focus on what's best for the community
